En esta presentación se hará la introducción al desarrollo de comparadores en el lenguaje de programación Kotlin. Aquí veremos los elementos de programación que permiten realizar todo tipo de comparaciones entre objetos en el lenguaje y el uso de un conjunto especial de objetos, llamados Comparators para agregar esquemas especiales de comparación entre objetos.

Orden Natural

Como todo lenguaje de programación que se tiene, Kotlin utiliza los siguientes operadores de comparación:

Tabla 1. Operadores de Comparación
Operador Significado Ejemplo

<

Menor que

12 < 10

< =

Menor o igual a

'c' < = 'd'

>

Mayor que

5 > -2

>=

Mayor o igual a

3.14 >= 2.71

==

Igual a

18 == 18

!=

Diferente a

-5.0 != 9.12

Estos operadores se encuentran ya definidos para los tipos básicos del lenguaje Kotlin como lo son los tipos Int, Double, Boolean y Char. Todos entendemos claramente como se realiza la comparación de los tipos numéricos, pero con los caracteres (el tipo Char) hay que tener en cuenta que de forma arbitraria, se definió un ``orden'' para las letras en el mundo computacional. Inicialmente se usó la tabla ASCII = American Standard Code for Information Interchange que se presenta en la tabla a continuación:

Figura 1. Tabla ASCII

Aunque actualmente se utiliza los códigos UNICODE en vez de ASCII, debemos indicar que UNICODE incluye a ASCII dentro de si. En la tabla anterior podemos observar que cada letra que el computador puede manejar tiene un código numérico asociado, que definen el orden de comparación de las letras en el mundo de los computadores. Podemos observar, por ejemplo, que los caracteres especiales son menores qe el resto de caracteres del alfabeto, y que las mayúsculas (que aparecen primero y tienen un código numérico inferior) son menores que las mayúsculas.

A partir de la tabla, podemos afirmar que la siguiente lista de comparaciones obtienen un valor de verdad true o verdadero:

  • 'A' < 'a'

  • '{' >= '('

  • '3' < 'C'

  • '^' > 'K'

Y las siguientes comparaciones son falsas:

  • 'Z' < 'm'

  • '&' >= '1'

Comparación de Cadenas

Las cadenas de caracteres o String también se pueden comparar en Kotlin utilizando los operadores de comparación presentados anteriormente. Hay que tener en cuenta, igual que con las letras que pertenecen al tipo Char, al comparar cadenas hay que tener el orden de las diversas letras que componen el string. También sucede, como con los caracteres, que las letras minúsculas son diferentes a las mayúsculas, y es necesario tener mucho cuidado con eso.

A partir de esto, las siguiente comparaciones son ciertas:

  • "casa" == "casa"

  • "ALTO" != "alto"

  • "2" > "12"

  • "dedo" < "mesa"

  • "resto" >= "ZAPATO"

Sin embargo, surge con mucha frecuencia, la necesidad de realizar comparaciones entre cadenas de caracteres pero sin tener en cuenta el tamaño de las letras (el case en inglés), es decir, sin tener en cuenta que las letras estén en mayúsculas o minúsculas. Para lograr esto, el lenguaje de programación Kotlin nos proporciona el método equal para los String. Este método permite saber si dos objetos de la clase String son iguales. Es decir, es lo mismo realizar una comparación con == que con .equals

Uso del equals
if (palabra1 == palabra2 || (palabra1.equals(palabra2)))

Pero si queremos comparar las palabras sin tener en cuenta la diferencia entre mayúsculas y las minúsculas, debemos usar el método equals pero le agregamos un segundo parámetro o dato de entrada con un valor en true y de esa manera, la comparación de igualdad no tendrá en cuenta el tamaño de las letras.

Equals sin tener en cuenta el tamaño de las letras
val s1 = "abcde"
val s2 = "AbCdE"

if (s1.equals(s2, true)) {
    println("Son iguales")
}
else {
    println("No son iguales")
}

En el anterior fragmento de código, vemos que las variables s1 y s2 contienen las mismas letras, pero escritas usando mayúsculas o minúsculas. Si ejecutamos el programa, obtendremos por pantalla la frase Son iguales, ya que usamos el método equals con la opción true para que no tenga en cuenta esas diferencias en el tamaño de las letras.

Importante
la operación == siempre compara teniendo en cuenta el tamaño de las letras.

Objetos Comparable

Kotlin permite crear objetos que se pueden comparar utilizando los operadores de comparación que hemos presentado anteriormente. Muchas veces, tener objetos comparables nos permitirá ordenar colecciones o estructuras de datos conteniendo esos objetos de una forma fácil y rápida.

Para poder tener objetos comparables, debemos definir el criterio de comparación de los objetos. Este criterio de comparación indica el atributo o grupo de atributos de la clase que deben manejarse para determinar si dos objetos de la clase son iguales, diferentes o si uno es mayor o menor que otro.

Una vez definido este criterio de comparación, los siguientes pasos para crear clases de objetos comparables son los siguientes:

  • Implementar la interface Comparable para la clase de objetos que deseamos comparar.

  • Escribir el método compareTo(otro), el cual teniendo en cuenta el criterio de comparación:

    • retorna cero si el objeto this es igual al otro

    • retorna un número negativo si el objeto this es menor que el otro

    • retorna un número positivo si el objeto this es mayor que el otro

Primer ejemplo

Supóngase que vamos a implementar la clase Perro, el cual tendrá tres atributos: el nombre del perro, la raza del perro y la edad del perro. Para comparar dos perros, se utilizará como criterio único de comparación al nombre del perro. Es decir, para saber si dos perros son iguales, basta con ver sus nombres, el resto de los atributos no son importantes en la tarea de comparar perros.

Entonces, ya definido el criterio de comparación, el método de comparación que debemos escribir para hacer de los objetos Perro comparables, tendrá el siguiente comportamiento:

  • Si los nombres de los perros son iguales, se retornará un cero

  • Si el nombre del primer perro (objeto this) es menor que el nombre del segundo perro, debemos retornar -1

  • Y si el nombre del primer perro es mayor que el nombre del segundo perro, debemos retornar 1.

La clase Perro comparable quedaría entonces de la siguiente manera:

Clase Perro Comparable
data class Perro(var nombre: String, var raza: String, var edad: Int) : Comparable<Perro> {
    override fun compareTo(other: Perro): Int {
        if (this.nombre == other.nombre) {
            return 0  // Nombres iguales, retornar cero
        }
        else if (this.nombre < other.nombre) {
            return -1
        }
        else {
            return 1
        }
    }
}

En este ejemplo observamos la estructura que deben tener los comparadores. Note que cómo el criterio de comparación es el nombre del perro, solo realizamos las comparaciones con ese atributo. Una vez ya definida la clase Perro como comparable, podemos realizar comparaciones comunes entre dos objetos de esta clase. Por ejemplo, las siguientes instrucciones producen un resultado de verdad de verdadero o true:

Uso de la clase Comparable
// Vamos a crear tres perros
val p1 = Perro("chacho", "pastor", 11)
val p2 = Perro("junior", "fox", 6)
val p3 = Perro("angel", "chihuahua", 9)
// Las siguientes sentencias son verdaderas
assertTrue { p1 != p2 }
assertTrue { p3 < p1 && p3 < p2 }
assertTrue { p2 >= p1 }

Segundo Ejemplo: criterio de comparación multi-atributo

Cuando el criterio de comparación consta de muchos atributos, el método compareTo debe tener en cuenta esos atributos múltiples. De todos los atributos que hacen parte del criterio de comparación, uno de ellos será el principal, otro será el secundario, y así sucesivamente. El esquema de estos comparadores tienen la siguiente forma:

  • Retorne un número negativo si el atributo principal de this es inferior al atributo principal del otro

  • Retorne un número positivo si el atributo principal de this es mayor que el atributo principal del otro objeto.

  • Si el atributo principal de ambos objetos son iguales, hacer el mismo proceso, pero con el atributo secundario, y así sucesivamente.

Suponga que vamos a escribir una versión comparable de un objeto Triángulo, que se define a partir de la base y la altura del mismo. El criterio de comparación que se escogió fue: primero la altura, y luego la base del triángulo. El siguiente código nos presenta la forma de construir esta clase comparable:

Un triángulo comparable
data class Triángulo(var base: Double, var altura: Double) : Comparable<Triángulo> {
    override fun compareTo(other: Triángulo): Int{
        // Primero vamos con el atributo principal
        if (this.altura < other.altura) {
            return -1
        }
        else if (this.altura > other.altura) {
            return 1
        }
        else {
            // Atributo principal iguales, vamos con el atributo secundario
            if (this.base < other.base) {
                return -1
            }
            else if (this.base > other.base) {
                return 1
            }
            else {
                return 0
            }
        }
    }
}

Funciones de ayuda

Kotlin nos proporciona un grupo de funciones útiles en la tarea de construir objetos comparables. Debemos partir del hecho que todos los tipos básicos del lenguaje son comparables. Es decir, las variables de tipo Int, Double, Char, Boolean o String son comparables y por lo tanto todas las variables que pertenecen a estos tipos poseen un método .compareTo que puede ser usado para facilitar el trabajo de crear objetos comparables más complejos.

Por ejemplo, vamos a crear la clase Planeta, la cual tendrá dos atributos: un código (número entero) y el nombre. Esta clase será comparable y usaremos el código como único criterio de comparación. El código de la clase es el siguiente:

Clase planeta
data class Planeta(val código: Int, nombre: String) : Comparable<Planeta> {
    override fun compareTo(other: Planeta): Int{
        // Usamos el método compareTo del código para hacer
        // todo el trabajo sucio del if que teníamos antes.
        return this.código.compareTo(other.código)
    }
}

Otra opción al compareTo es utilizar la función de ayuda llamada compareValues. Esta función recibe dos datos de entrada de cualquier tipo básico, y retorna cero si ambos datos son iguales, un número positivo, si el primer dato es mayor al segundo, y un número negativo si el primer dato es menor que el segundo. Es decir, hace justo lo que necesitamos. Veamos como usarlo.

Supóngase que vamos a hacer la clase Punto como comparable, y el criterio de comparación que usaremos será la distancia al origen del punto. El código completo de la clase es el siguiente:

La clase Punto comparable
data class Punto(var x: Double, var y: Double) : Comparable<Punto> {
    // Método interno que halla la distancia al origen del punto
    fun distOrigen(): Double = sqrt(x.pow(2) + y.pow(2))

    // Método que compara el punto this y el punto other
    override fun compareTo(other: Punto): Int{
        return compareValues(this.distancia(), other.distancia())
    }
}

Las anteriores dos opciones presentadas solo funcionan cuando el criterio de comparación consta de un solo único atributo, pero no son útiles cuando el criterio consta de múltiples atributos. Pero Kotlin nos proporciona otras funciones para ayudarnos a crear comparaciones con la participación de muchos atributos. Esta función que nos ayuda para facilitarnos la tarea de crear un compareTo de múltiplos atributos es compareValuesBy. Esta función recibe los dos objetos a comparar (this y other), y una lista de selectores, que corresponde a los atributos o métodos que utilizaremos como criterios de comparación. Un selector tiene la siguiente forma: Clase::atributo o Clase::metodo, para indicar el nombre del atributo y del método de la clase que usaremos como criterio de comparación.

En el siguiente ejemplo vamos a crear una clase comparable llamada Película, la cual tiene dos atributos: el nombre de la película, el año en que se estrenó y el país al que pertenece la película. Vamos a indicar que el criterio de comparación principal de esta clase será el nombre de la película y luego usaremos el año de estreno. El código completo de la clase, pero usando la función de utilidad compareValuesBy es el siguiente:

La clase Película comparable
data class Película(var nombre: String,
                    var añoEstreno: Int,
                    var país: String) : Comparable<Película> {
    // El método que compara películas. Criterio: nombre y año de estreno
    override fun compareTo(other: Película): Int{
        return compareValuesBy(this, other, Película::nombre, Película::añoEstreno)
    }
}

Comparadores

Un comparador es un tipo especial de objetos cuya única misión en la vida es proporcionar la comparación entre dos objetos de la misma clase. Aunque los objetos comparables permiten comparar objetos de la misma clase, utilizando los operadores comunes y corrientes del lenguaje Kotlin, como son ==, !=, < y >. Sin embargo, existen situaciones donde necesitamos más de una forma de comparar objetos de la misma clase y es por está razón que nace en nuestra ayuda los objetos comparadores.

Los objetos comparadores deben implementar la interface Comparator y por esto deben, obligatoriamente, reescribir el método compare, que tiene un comportamiento idéntico al método compareTo que hemos presentado anteriormente, con la principal diferencia que compare recibe los dos objetos a comparar.

Por ejemplo, supóngase que tenemos la clase Producto, que posee un nombre y un precio. Vamos a hacer esta clase comparable utilizando el nombre como único criterio de comparación. Esta clase quedaría de la siguiente manera:

La clase Producto comparable
data class Producto(var nombre: String, var precio: Double) : Comparable<Producto> {
    // Compara el producto this contra el producto other usando el nombre
    override fun compareTo(other: Producto): Int {
        return compareValues(this.nombre, other.nombre)
    }
}

Ahora deseamos crear otro comparador para productos, pero usando el precio como criterio de comparación. Para crear un comparador, Kotlin nos proporciona una serie de funciones de ayuda para hacer la creación de estos objetos mucho más rápido. La función que vamos a utilizar se llama compareBy. Esta función recibe los selectores de atributos, es decir, la lista de atributos o métodos que utilizaremos como criterio de comparación para el comparador. El código quedaría así:

Un comparador de productos
val comparadorPrecio: Comparator<Producto> = compareBy(Producto::Precio)

// Ahora, podemos usar ese comparador para ver, entre dos productos, cual es el mas caro
if (comparadorPrecio.compare(prod1, prod2) > 0) {
    println("El producto 1 es el más caro")
}
else {
    println("El producto 2 es el más caro")
}

Note las dos etapas del trabajo con un comparador. La definición del mismo (usando la función compareBy) y seleccionando los atributos. Y la otra etapa es cuando necesitamos comparar los dos objetos a través del criterio de comparación que hace parte del comparador. Para eso usamos entonces el método compare del objeto comparador y le pasamos como parámetros o datos de entrada, los dos objetos que deseamos comparar.

Ya que el compare funciona igual que el compareTo, si queremos determinar que los dos objetos son iguales, solo tenemos que comprobar que el valor retornado sea cero. Si queremos determinar si el primer objeto es superior al segundo (de acuerdo a los criterios de comparación), solo debemos comprobar si el método compare retorna un número positivo, y si finalmente queremos saber si el primer objeto es inferior al segundo, debemos comprobar que el método compare retorne un número negativo.

Otra función de ayuda que tenemos para crear comparadores es reversed(), que permite invertir el orden del comparador. Es decir, retornará un valor positivo si el primer objeto es inferior al segundo (acuérdese que en el funcionamiento normal, debería obtenerse un valor negativo) y retornará un valor negativo si el primer objeto es superior al segundo.

Ejemplo de reversed
val comp1: Comparator<Perro> = compareBy(Perro::nombre).reversed()

Finalmente, otra función de ayuda que nos ofrece Kotlin se conoce con el nombre de thenBy. Este método permite extender el criterio de comparación a nuevos atributos y métodos. Se usa en conjunción con el compareBy para proporcionar una lectura más natural del comparador. Es una forma más elegante de escribir criterios de comparación multi-atributos.

Por ejemplo, si queremos crear un comparador de perros, que utilice como primer criterio el nombre del perro y como segundo criterio la edad del perro, podriamos escribirlo usando solamente el método compareBy o usando una combinación de compareBy y thenBy.

Ejemplo de thenBy
val comp1: Comparator<Perro> = compareBy(Perro::nombre, Perro::edad)

// El mismo comparador, pero usando thenBy
val comp2: Comparator<Perro> = compareBy(Perro::nombre).thenBy(Perro::edad)

Un ejemplo del uso de los comparadores sería la siguiente función externa, que recibe una lista de objetos de la clase triángulo y que busca el triángulo más pequeño, teniendo en cuenta el siguiente de criterio de comparación: primero tenga en cuenta la base del triángulo y luego la altura del triángulo. El código de la función es la siguiente:

Ejemplo de hallar el menor
fun menorTriangulo(triangulos: IList<Triangulo>): Triangulo {
    var menorTriangulo = triangulos[0]   // El menor es el primero
    val comp: Comparador<Triangulo> = compareBy(Triangulo::base, Triangulo::altura)

    for (t in triangulos) {
        if (comp.compare(t, menorTriangulo) < 0) {
            // Si t es menor que el menor triangulo, cambie el menor triangulo
            menorTriangulo = t
        }
    }
    return menorTriangulo
}

En el ejemplo anterior podemos observar el uso que tenemos que darle a los comparadores cuando necesitamos saber si un objeto es menor que otro.
